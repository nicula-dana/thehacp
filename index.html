Overview

HACP (Human-Agent Collaboration Protocol) is a modular, open protocol designed to enable effective, high-trust collaboration between humans and AI agents in remote, asynchronous, and dynamic work environments.

Core Principles

Hybrid Work as Default

Work is distributed across humans and autonomous agents, each contributing unique capabilities.

Roles as Functions, Not Titles

Responsibilities are dynamically claimed, not statically assigned.

Flow over Sprints

Work moves in responsive microloops triggered by real-world signals, not fixed time boxes.

Transparency by Design

Every decision or contribution is traceable and auditable, regardless of origin (human or AI).

Cognitive Load Awareness

Attention is treated as a finite asset, managed proactively via system-level protocols.

Trustworthy Autonomy

AI agents must explain their actions and meet trust thresholds for influence.

Protocol Structure

1. Signals

  • New data or insight emerges
  • A decision is required
  • A role token is unclaimed

2. Loops

  • Discovery Loop: Insight synthesis and framing
  • Alignment Loop: Consensus building
  • Delivery Loop: Build/test/validate

3. Role Tokens

Initiator (Human)Proposes new work based on needs or insight
Decider (Human)Holds decision authority within scope
Synthesizer (Human)Integrates insight into deliverables
Validator (Human)Confirms or rejects outcomes
Narrator (Human)Documents progress and lessons
Signal Scanner (Agent)Monitors for triggers
Orchestrator (Agent)Coordinates agents and people
Implementer (Agent)Executes tasks autonomously
Shadow Logger (Agent)Records for audit
Explainer (Agent)Makes AI reasoning understandable

  • Priority Graph: Non-linear backlog sorted by urgency
  • Shadow Logs: Agent suggestion and interaction records

5. Boundaries

  • Ethical: fairness, transparency
  • Cognitive: attention and workload limits
  • Technical: data and tools

Advanced Concepts

Dynamic Role System

Roles can be claimed dynamically by any contributor (human or AI) with the capability, creating modular micro-governance.

Agent-Oriented Workflows (AOWs)

  • Trigger conditions
  • Required tokens
  • Success criteria

Cognitive Load Contracts (CLCs)

Each participant defines their attention availability; work is routed accordingly.

Trust Ledger

Tracks agent reliability and influence. Actions are auditable and explainable.

Implementation Guide

  • Start Small: Pilot a loop with HACP roles
  • Visualize Everything: Use a live SAG or dashboard
  • Log Transparently: Record all agent interactions
  • Reflect Often: Run mini retrospectives per loop
  • Scale Modularly: Add roles and flows as needed
